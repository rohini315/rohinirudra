AudioDrops

Augmented Audio Experience that enables bands, brands, and fans to personalize space and discover music anywhere.

Company

AudioDrops

My role

UX Consultant | User Testing, Wireframes

Duration

3 weeks (February 2018)

Process

Background and brief

I was hired as a UX/UI designer to help with the next phase of the design process for AudioDrops' mobile app. At first it was a daunting task being the only designer and starting off where the last designer left off, but I was willing to take that challenge. My brief was to re-design the mobile interface as well as add some new features to make the app more user friendly. The company's main goal was to try and get users to walk around and discover new music along the way.

Understood the new business objectives and uncovered users pain points.

To gain more insight, I interviewed the Founders and CEO on the app/companies history. The UI had gone through several major changes in the past few years. They were in the process of speaking with different music tourism boards in various cities, and wanted to create a universal way to engage individuals users, as well as larger organizations.

Uncovered major UX/UI problems

Before I conducted any user testing, I noticed a lot of problems being a new user myself. I wanted to see if the problems I had with the app would match up to what other new users felt.

Conducted multiple user tests to pinpoint painpoints

I conducted over ten user tests where I had each user download the app, create an account and have them drop their first song. I then asked each user to complete a series of tasks and asked them to think out loud.

Some of the pain points users had were:

  • Users had a difficult time figuring out how to navigate to another user’s profile page and follow them.
  • Users didn’t understand what the ‘collection’ tab was for.
  • Unsure if drop was successful after clicking the drop button.
  • Did not understand the difference between Your Set’s Drops vs ____’s drops in the sidebar menu.
  • Once connected to Spotify- users were unsure if it was properly connected - Before clicking the sidebar menu, thought the tabs would be related to account settings
  • Thought they had to log into Spotify before searching for a song- unclear directions
  • Wanted to choose what songs would be added to their collection
  • Some thought the main ‘drops’ tab would recenter the map
  • Some users had a difficult time finding the logout button

After conducting each test I asked for overall feedback. Many of the users shared the same thoughts as me: Very interesting application, but UX needs to be more clear.

Brainstorming

I shared all the data I had collected with the founders and the developers. We all sat down and prioritized what user pain points we should tackle first.

My next steps were to start with sketching out some pen and paper wireframes that addressed some of the apps major issues. My plan is to keep working on pen and paper sketches, doing some more user testing and then translating those into lo-fi mockups in sketch.

Reflections

I had a great time at AudioDrops and working with the team. I was offered a position at antoher company and couldn't continue my work here, but I was happy to help out!
